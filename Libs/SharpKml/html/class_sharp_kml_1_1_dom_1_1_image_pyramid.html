Detailed Description

Specifies a hierarchical set of images, each of which is an increasingly lower resolution (towards the top of the pyramid).

OGC KML 2.2 Section 11.6

Each image in the pyramid is subdivided into tiles so only the portions in view are loaded.

The pixel size of the original image is specified by Width and Height. The width and height can be any size and do not need to be a power of 2. You can fill out the remaining pixels with blank pixels (as described in Section 11.6.3)

Tiles must be square, and the TileSize must be a power of 2. A tile size of 256 (the default) or 512 is recommended.

See Section 11.4.3 for handling large images and Section 11.6.3 for creating an Image Pyramid.
