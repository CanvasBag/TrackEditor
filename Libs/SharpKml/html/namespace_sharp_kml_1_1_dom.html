◆ ColorMode

enum SharpKml.Dom.ColorMode
strong

Specifies the color mode for a graphic element.

OGC KML 2.2 Section 16.7

If the value is Random then the color components are modified as follows:

  • If a single color component is specified (for example Color32.Red equals 255), random color values for that one component will be selected. In this case, the values would range from 0 (black) to 255 (full red).
  • If values for two or for all three color components are specified, a random linear scale is applied to each color component, with results ranging from black to the maximum values specified for each component.
  • The opacity of a color (Color32.Alpha) is never randomized.
Enumerator
Normal 

Use a single color value.

Random 

Use a random color value.

Specifies how a Feature and its contents shall be displayed as items in a list view.

OGC KML 2.2 Section 16.15

Enumerator
Check 

The Feature's visibility is tied to its item's checkbox.

RadioFolder 

Only one of the Container's items shall be visible at a time.

CheckOffOnly 

Use a normal checkbox for visibility but do not display the Container's children in the list view.

A checkbox allows the user to toggle visibility of the child objects in the viewer.

CheckHideChildren 

Prevents all items from being made visible at once.

The user can turn everything in the Container off but cannot turn everything on at the same time. This setting is useful for Container's containing large amounts of data.
